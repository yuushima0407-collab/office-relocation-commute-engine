# CEST v0.3.1 Frontend 仕様

**担当**: pachi
**前提**: v0.3.1 backend がマージ済みであること
**ステータス**: 未着手

---

## 概要

v0.3.1 バックエンドの新フィールドに対応し、UX を改善する。
画面をタブ構成にして、ユーザーの操作導線を整理する。

---

## 1. 画面構成のタブ化

現状は全セクションが縦に並んでいて見通しが悪い。タブで分割する。

```
┌─────────────────────────────────────────────────┐
│  CEST v0.3.1 - オフィス配置シミュレーション       │
├─────────────────────────────────────────────────┤
│  [入力] [結果一覧] [地図] [比較] [感度分析]      │  ← タブ
├─────────────────────────────────────────────────┤
│                                                  │
│  （選択中のタブの内容が表示される）               │
│                                                  │
└─────────────────────────────────────────────────┘
```

### タブ構成

| タブ | 内容 |
|------|------|
| 入力 | オフィス候補・社員データ・設定（将来実装） |
| 結果一覧 | 制約サマリ + 散布図 + コンボカード一覧 |
| 地図 | 選択した案の通勤マップ（常時表示、案選択で更新） |
| 比較 | 2案の比較パネル |
| 感度分析 | 感度分析の impacts テーブル |

---

## 2. 感度分析の v0.3.1 対応

バックエンドの感度レスポンスが変わるため、表示を更新する。

### 旧形式（v0.3）— 廃止
```json
{ "ranking_stable": true, "details": [{ "ranking_changed": false }] }
```

### 新形式（v0.3.1）
```json
{
  "summary": "...",
  "details": [{
    "parameter": "last_mile_minutes +5min",
    "description": "...",
    "impacts": [
      { "combo_name": "品川+渋谷", "original_p95": 67, "changed_p95": 72, "diff": 5 }
    ]
  }]
}
```

### 表示
- 「安定/不安定」バッジ → **廃止**
- パラメータごとに impacts テーブル（案名 / 変更前 / 変更後 / 差分）を表示
- summary をテーブルの上に表示

## 3. 通勤費の表示

### コンボカード
```
月額賃料:     480万円
通勤費(推定): 142万円
───────────────────
合計:         622万円
```
- `total_commute_cost_estimated` が true なら「(推定)」を付記

### 散布図
- X軸に「合計コスト（賃料+通勤費）」オプションを追加
- 既存: total_rent / rent_per_person
- 追加: total_cost（= total_cost_jpy_month）

### 比較テーブル
- 通勤費・合計コストの行を追加

---

## 4. 散布図の改善

- 常時ラベル表示 → **ホバーツールチップ**に変更（ラベル重なり解消）
- Y軸に「週間通勤時間」オプション追加: p95 × 2 × 出社日数
- 出社日数スライダー変更で散布図が即座に更新（API再呼出し不要）

---

## 5. 比較テーブルに分布行追加

```
30分未満:  45人 vs 38人
30-60分:   52人 vs 60人
60分以上:   8人 vs  7人
```

## 6. 地図の常時表示

現状: コンボをクリックしないと地図が出ない（気づかない人がいる）
改善: 地図タブに移動し、案を選択すると地図が更新される導線にする
