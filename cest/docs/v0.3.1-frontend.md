１# CEST v0.3.1 Frontend

**担当**: pachi
**前提**: v0.3.1 backend がマージ済み（PR #6）
**ステータス**: 未着手

---

## 背景

v0.3.1 バックエンドで以下が変わった。フロントをこれに対応させたい。

### バックエンドの変更点
- 各案に `total_commute_cost_jpy_month`, `total_cost_jpy_month`, `cost_per_capacity` 等が追加
- `total_commute_cost_estimated` (bool) で推定値かどうか判別可能
- 感度分析のレスポンス形式が変更（後述）

---

## 解決したい課題

1. **画面が縦に長すぎて見通しが悪い** — 全セクションが1ページに並んでいて、下の方が埋もれる
2. **地図に気づかない人がいる** — コンボをクリックしないと表示されない
3. **通勤費が見えない** — バックエンドは返しているがフロントに未反映
4. **散布図のラベルが重なる** — 候補が多いと読めない
5. **感度分析が旧形式のまま** — バックエンドが impacts 形式に変わったので対応が必要

---

## 対応してほしいこと

### 必須

**A. 画面構成の改善**
- 縦並びの解消。タブ・ステップ・アコーディオン等、方法は任せる
- ユーザーが「入力 → 結果確認 → 詳細比較」と自然に進める導線

**B. 通勤費の表示**
- コンボカード・比較・散布図に通勤費と合計コストを反映
- 推定値の場合はその旨が分かるようにする
- 使うフィールド: `total_commute_cost_jpy_month`, `total_cost_jpy_month`, `total_commute_cost_estimated`

**C. 感度分析の v0.3.1 対応**
- 旧形式の `ranking_stable` / `ranking_changed` は廃止
- 新形式:
```json
{
  "summary": "ラストマイル+5分で...",
  "details": [{
    "parameter": "last_mile_minutes +5min",
    "description": "ラストマイルが5分増えた場合の影響",
    "impacts": [
      { "combo_name": "品川+渋谷", "original_p95": 67, "changed_p95": 72, "diff": 5 }
    ]
  }]
}
```
- 表示方法は任せる

**D. 地図の導線改善**
- 現状「クリックしないと出ない」のを、もっと自然に見れるようにする
- 方法は任せる（常時表示、タブ化、案選択で自動表示、等）

### できれば

**E. 散布図のラベル改善**
- ラベル重なりの解消（ホバーツールチップ化など）

**F. 比較テーブルに通勤分布を追加**
- 30分未満 / 30-60分 / 60分以上の人数

**G. CSVアップロード UI**
- `POST /parse-csv/upload` を使ったファイルアップロード機能
- CSV/TSV/Excel (.xlsx) 対応
- デモモードでも CSV/TSV はクライアント側で解析可能
