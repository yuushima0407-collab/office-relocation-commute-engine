## 2. 通勤費推定エンジン

JR東日本 IC運賃テーブルを用いた通勤費推定を追加。

### 推定ロジック
- 通勤時間 → 距離推定（平均速度 40km/h）→ JR IC運賃テーブルで片道運賃取得
- 月額 = 片道 × 2 × 出社日数 × 4.33週 × 定期割引(0.85)
- CSVに通勤費の実費データがあれば推定より優先

### Settings 追加
```json
{
  "commute_cost_policy": "full",
  "commute_cost_cap_jpy_month": null
}
```
- `"full"`: 全額会社負担（デフォルト）
- `"capped"`: 上限あり（cap値を参照）
- `"ignore"`: 通勤費を考慮しない

### レスポンス追加フィールド
```json
{
  "total_commute_cost_jpy_month": 1420000,
  "total_commute_cost_estimated": true,
  "total_cost_jpy_month": 6220000,
  "total_capacity": 200,
  "cost_per_capacity": 31100
}
```

**新規ファイル**: `fare_estimator.py`
**変更ファイル**: `combination.py`, `request.py`

## 4. パレートフロンティア 3軸化

旧: (p95_trip_minutes, total_rent_jpy_month) の2軸
新: (avg_trip_minutes, total_cost_jpy_month, total_capacity) の3軸

- avg に変更: p95 は外れ値の影響を受けやすく、平均の方が全体傾向を反映
- total_cost: 賃料+通勤費の合計で評価
- capacity: 収容人数も考慮（大きい方が有利）

**変更ファイル**: `combination.py`

---

## 5. 感度分析の刷新

旧: `ranking_stable` (bool) + 各パラメータの `ranking_changed` (bool)
新: 具体的な数値影響を impacts テーブルで返す

### 新レスポンス形式
```json
{
  "summary": "ラストマイル+5分で最大影響は品川+渋谷案（p95: 67→72分）。出社5日では週間通勤が最大600分変化。",
  "details": [
    {
      "parameter": "last_mile_minutes +5min",
      "description": "ラストマイルが5分増えた場合の通勤最長(p95)への影響",
      "impacts": [
        {"combo_name": "品川+渋谷", "original_p95": 67, "changed_p95": 72, "diff": 5}
      ]
    },
    {
      "parameter": "出社日数 3日→5日",
      "description": "出社日数が変わった場合の週間通勤時間への影響",
      "impacts": [
        {"combo_name": "品川+渋谷", "original_weekly_minutes": 180, "changed_weekly_minutes": 300, "diff": 120}
      ]
    }
  ]
}
```

**変更ファイル**: `combination.py`, `pipeline.py`
