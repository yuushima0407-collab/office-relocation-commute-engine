# CEST v0.3.3 仕様

**ステータス**: ドラフト（v0.3.1 完了後に着手）

---

## 概要

v0.3.1 の「候補を絞る」ツールから、**「意思決定を支援する」ツール**への進化。
パレートで残った案を、関係者が納得して選べるようにする。

---

## 新機能一覧

| # | 機能 | 目的 | 担当 |
|---|------|------|------|
| 1 | 部署別影響ビュー | 合意形成の支援 | backend + frontend |
| 2 | Before/After比較 | 現状との差分を明確に | backend + frontend |
| 3 | コスト↔通勤スライダー | パレートから1案に絞る支援 | frontend のみ |
| 4 | What-If シナリオ | 将来変化への耐性確認 | frontend のみ |
| 5 | 意思決定レポート（PDF） | 経営層への提案資料 | frontend のみ |
| 6 | 感度分析の簡略化 | 不要な複雑さの除去 | backend + frontend |

---

## Backend 変更

### 1. 部署別影響ビュー

`all_combinations[]` の各案に `department_breakdown` を追加:

```json
{
  "department_breakdown": [
    {
      "group": "営業部",
      "count": 30,
      "avg_trip_minutes": 42,
      "p95_trip_minutes": 68,
      "assigned_office": "渋谷オフィス",
      "commute_cost_jpy_month": 450000
    }
  ]
}
```

対立ポイントの自動検出:
```json
{
  "conflict_alerts": [
    {
      "type": "department_gap",
      "message": "営業部(p95: 68分)と経理部(p95: 45分)で23分の格差があります",
      "severity": "warning"
    }
  ]
}
```

### 2. Before/After 比較

`settings.baseline` オブジェクトを追加（候補に含まれない現オフィスも指定可能）:

```json
{
  "settings": {
    "baseline": {
      "office_id": "current_office",
      "name": "現オフィス（丸の内）",
      "nearest_station_id": "tokyo",
      "last_mile_minutes": 5,
      "rent_jpy_month": 8000000,
      "capacity_people": 90
    }
  }
}
```

各案に `vs_baseline` を追加:
```json
{
  "vs_baseline": {
    "avg_trip_change": 5.2,
    "p95_trip_change": -3.0,
    "rent_change": -2000000,
    "commute_cost_change": 150000,
    "total_cost_change": -1850000,
    "worse_count": 35,
    "better_count": 60,
    "unchanged_count": 10
  }
}
```

### 3. 感度分析の簡略化

`compute_sensitivity_v3` → `compute_closeness_warning` に置換:

```json
{
  "sensitivity": {
    "close_pairs": [
      {
        "combo_a": "k2_combo_1",
        "combo_b": "k2_combo_3",
        "cost_diff_jpy_month": 45000,
        "avg_trip_diff_minutes": 2.1,
        "message": "案Aと案Cは僅差です"
      }
    ]
  }
}
```

### 変更ファイル

| ファイル | 変更内容 |
|---------|---------|
| `request.py` | `settings.baseline` オブジェクト追加 |
| `combination.py` | `department_breakdown`, `vs_baseline`, `conflict_alerts` 追加 |
| `combination.py` | `compute_sensitivity_v3` → `compute_closeness_warning` |
| `pipeline.py` | baseline 計算の追加 |

## Frontend 変更

### コスト↔通勤スライダー（フロントのみ）
パレート最適案に重み付けスコアを計算し推奨案をハイライト。

### What-If シナリオ（フロントのみ）
入力を加工して `/evaluate` を再リクエスト。新APIは不要。
- 部署の人数増減
- 候補オフィスの除外

### 意思決定レポート（フロントのみ）
html2pdf.js 等で PDF 生成。バックエンド変更不要。

---

## 実装順序

| Phase | 内容 | 担当 |
|-------|------|------|
| 1 | Before/After + 部署別影響 | backend → frontend |
| 2 | コスト↔通勤スライダー + 感度簡略化 | backend → frontend |
| 3 | What-If シナリオ | frontend のみ |
| 4 | PDF レポート | frontend のみ |

---

## 実装しないもの

| 項目 | 除外理由 |
|------|----------|
| 路線データ拡充 | 現行で十分 |
| 乗り換えペナルティ | 駅ごとに異なり精度出ない |
| 不動産API連携 | プロダクト化フェーズの話 |
| 複数シナリオ並列比較 | What-If で代替 |
